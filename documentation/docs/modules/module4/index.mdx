### Update the code to consume agent from Foundry

At this point, our bot is not functional yet as it does not do much. We need to update the code to consume our Microsoft Foundry agent using the Foundry SDK.
Behind the scenes, AI Foundry Agent Service uses the OpenAI [Responses API](https://platform.openai.com/docs/api-reference/responses) to interact with the agent **even though the agent is not using an OpenAI model (e.g., DeepSeek models)**. This means you can use the same code/properties (except for some specificities) described in the OpenAI documentation but with the AI Foundry endpoints.

1. Replace the code in the `agent.ts` file with the following content:

    ```typescript
    import { AgentApplication, Storage, TurnContext, TurnState } from "@microsoft/agents-hosting";
    import { ClientSecretCredential, TokenCredential } from "@azure/identity";
    import { ActivityTypes } from "@microsoft/agents-activity";
    import { AIProjectClient } from "@azure/ai-projects";
    import config from "./config";

    export class CustomAgent extends AgentApplication<TurnState> {

        constructor (storage: Storage) {
            super({
                storage: storage
            });

            this._onMessage = this._onMessage.bind(this);

            this.onActivity(ActivityTypes.Message, this._onMessage);
        }
    
        private async _onMessage(context: TurnContext, state: TurnState) {   

            // Non streaming mode
            await this.invokeAgent(context);
        }

        private async invokeAgent(context: TurnContext) {
            const foundryProject = new AIProjectClient(config.aiFoundryProjectEndpoint!, this.getCredential());

            const items: any[] = [{
                type: "message",
                role: "user",
                content: context.activity.text
            }];

            const client = await foundryProject.getOpenAIClient();
            const conversation = await client.conversations.create({
                items,
                agent_reference:{ 
                    type: "agent_reference",
                    name: config.agentName 
                } 

            } as any);

            const response = await client.responses.create(
                {
                    input: context.activity.text,
                    conversation: conversation.id,
                    agent_reference:{ 
                        type: "agent_reference",
                        name: config.agentName 
                    }
                }  as any
            );

            const lastOutput = response.output[response.output.length - 1] as { content?: Array<{ text?: string }> };
            await context.sendActivity(lastOutput.content?.[0]?.text ?? "");
        }

        public getCredential(): TokenCredential {

            return new ClientSecretCredential(
                config.tenantId!,
                config.clientId!,
                config.clientSecret!
            );
        }
    }
    ```

    In this code, we simply implement a new custom agent class `CustomAgent` as middleware between Microsoft 365 channels and the agent in Foundry. The `_onMessage` method will be called every time a user chats with the bot through configured channels (e.g., Teams or Copilot). We simply forward the user message to the Microsoft Foundry agent using the Azure Foundry SDK and OpenAI Responses API and return the response to the user via the original channel using the `context.sendActivity()` method.

    :::note
    For this example, the response is just the last message in the response output array. 
    :::

    :::tip
    The Azure Foundry SDK is not mandatory to interact with the agent. You can always use the Foundry [raw REST API](https://learn.microsoft.com/en-us/azure/ai-foundry/reference/foundry-project-rest-preview?view=foundry#create-conversation) if the SDK is missing some latest capabilities. 
    :::

    adapter = CloudAdapter(connection_manager=connection_manager)

    agent_app = AgentApplication[TurnState](
        storage=storage,
        adapter=adapter,
        **agents_sdk_config,
    )

            client = project_client.get_openai_client()

### Stream response from the agent

Now that we have basic input working, let's add streaming response capability to our agent to improve the user experience. The OpenAI Responses API supports streaming responses by setting the `stream` property to `true` when creating a response. We can leverage this capability to stream the response back to the user as it is being generated.

<Tabs defaultValue="typescript" values={[
{ label: 'TypeScript', value: 'typescript' },
{ label: 'Python', value: 'python' },

<TabItem value="typescript">

1. Create a new `invokeAgentWithStreaming` method and update the `_onMessage` method in the `agent.ts` to use it:

    ```typescript title="Streaming response"

    private async _onMessage(context: TurnContext, state: TurnState) {

        // Non streaming mode
        // await this.invokeAgent(context);

        await this.invokeAgentWithStreaming(context);
    }

    private async invokeAgentWithStreaming(context: TurnContext) {

        // Configure streaming response
        await context.streamingResponse.setDelayInMs(100);
        await context.streamingResponse.setGeneratedByAILabel(true);
        await context.streamingResponse.setFeedbackLoop(true);
        await context.streamingResponse.queueInformativeUpdate('Working on your answer...');

        const foundryProject = new AIProjectClient(config.aiFoundryProjectEndpoint!, this.getCredential());

        const items: any[] = [{
            type: "message",
            role: "user",
            content: context.activity.text
        }];

        const client = await foundryProject.getOpenAIClient();
        const conversation = await client.conversations.create({
            items,
            agent_reference:{ 
                type: "agent_reference",
                name: config.agentName 
            } 

        } as any);

        // Streaming mode: process events and send to Teams
        const responseStream = client.responses.stream(
        {
                input: context.activity.text,
                conversation: conversation.id,
                agent_reference:{ 
                    type: "agent_reference",
                    name: config.agentName 
                }
            }  as any
        );

        for await (const event of responseStream) {

            if (event.type === "response.output_text.delta") {
                if (event.delta && context) {
                    await context.streamingResponse.queueTextChunk(event.delta);
                }
            } 
            else if (event.type === "response.output_text.done") {
                
                if (event.text && context) {
                await context.streamingResponse.queueTextChunk(event.text);
                }

            } 
            else if (event.type === "response.completed") {
                await context.streamingResponse.endStream();
            }
        }
    }
    ```

    In this code, we combine the streaming capabilities of both the Microsoft 365 Agents SDK and the OpenAI Responses API (through the Foundry SDK `stream()` method). We configure the streaming response in the `context.streamingResponse` object and then process the events from the `responseStream` to send text chunks to the user as they are generated. You can use the `context.streamingResponse.setDelayInMs();` method to control the streaming pace as seen by the user.

1. Create a new `invoke_agent_streaming` function and update the `on_message` handler in `agent.py` to use it:

    ```python title="src/agent.py"
    import asyncio

    ...

    @agent_app.activity(ActivityTypes.message)
    async def on_message(context: TurnContext, state: TurnState):

        # Non streaming mode
        # await invoke_agent(context)

        # Streaming mode
        await invoke_agent_streaming(context)


    async def invoke_agent_streaming(context: TurnContext) -> None:
        try:

            context.streaming_response.set_generated_by_ai_label(True)
            await asyncio.sleep(0.1)

            context.streaming_response.set_feedback_loop(True)
            await asyncio.sleep(0.1)

            context.streaming_response.queue_informative_update("Working on your answer...")
            await asyncio.sleep(0.1)

            client = project_client.get_openai_client()

            conversation = client.conversations.create(
                items=[
                    {
                        "role": "user",
                        "content": context.activity.text,
                        "type": "message",
                    }
                ],
                extra_body={"agent_reference": {"type": "agent_reference", "name": config.ENV_FOUNDRY_AGENT_NAME}},
            )

            with client.responses.stream(
                model=config.ENV_FOUNDRY_AGENT_MODEL,
                input=context.activity.text,
                conversation=conversation.id,
                extra_body={"agent_reference": {"type": "agent_reference", "name": config.ENV_FOUNDRY_AGENT_NAME}},
            ) as stream:

                for event in stream:
                    event_type = getattr(event, "type", "")
                    if event_type == "response.output_text.delta":
                        delta = getattr(event, "delta", "")
                        if delta:
                            context.streaming_response.queue_text_chunk(delta)
                            await asyncio.sleep(0.05)

                    elif event_type == "response.completed":
                        await context.streaming_response.end_stream()

        except Exception as e:
            await context.send_activity(f"On message error. Details: {str(e)}")
    ```

    In this code, we combine the streaming capabilities of both the Microsoft 365 Agents SDK and the OpenAI Responses API (through the Foundry SDK `stream()` method). We configure the streaming response on the `context.streaming_response` object and then process the events from the `stream` to send text chunks to the user as they are generated. The short `await asyncio.sleep(...)` calls yield control back to the event loop so the SDK can flush the queued chunks progressively, giving the user a smooth streaming experience.
