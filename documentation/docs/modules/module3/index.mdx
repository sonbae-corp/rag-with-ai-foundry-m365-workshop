## Deploy a chat completion model

To be able to chat with our agent, we need to first deploy an appropriate chat completion AI model in our Microsoft Foundry project:

1. Navigate to your Microsoft Foundry project resource in the Azure portal (i.e., `project-ai-<id>`). You can also access the Foundry portal directly from [https://ai.azure.com/](https://ai.azure.com/) and select your project.

    <div className="flex justify-center mb-8">
        <img className="w-[70%]" src={FoundryPortalAccess}/>
    </div>

1. Click on the _"Build"_ tab and go to _"Models"_ to deploy a new **gpt-4.1 nano** base model with default settings:

    <div className="flex justify-center mb-10">
        <img className="w-[60%]" src={DeployCompletionModel}/>
        <img className="w-[40%] m-5" src={DeployGPT41Nano}/>
    </div>

    For the purpose of this workshop, we're going to use the `gpt-4.1 nano` model as it is a simple and economical model for basic RAG operations. However, depending on your use case, you might want to select a different model. You can filter available models by capabilities or intended tasks (e.g., chat, code generation, etc.).

    :::warning Completion model choice
    As mentioned in the [first module](../module1/#what-are-the-specificities-of-enterprise-environments), the choice of the model is critical as it directly impacts the cost and performance of your agent. **Using the latest and greatest is not always the best choice** as it tends to be much more expensive and sometimes overkill for your use case. As a best practice, always evaluate your AI needs (function calling, image generation, etc.) and test different models, starting from the cheapest to find the best fit. Make sure to use the Azure [AI Foundry Models](https://azure.microsoft.com/en-us/pricing/details/ai-foundry-models/microsoft/) (use [this link](https://azure.microsoft.com/en-us/pricing/details/cognitive-services/openai-service/) for OpenAI models) pricing page to see costs per model and also check if it is available in the **Canada East** or **Canada Central** region.
    :::

1. Once the model is deployed, you should see it in the list of available models in your Foundry project:

    <div className="flex justify-center mb-8">
        <img className="w-[70%]" src={GPTModelDeployed}/>
    </div>

## Create Azure AI Search connection

1. Create a new connection to your Azure AI Search resource created in [Module 2](../module2/#provision-azure-ai-search-instance-and-configure-index). To do this, from the Foundry portal, go to _"Operate"_→_"Admin"_→_&lt;Project&gt;_→_"Add connection"_→_"Azure AI Search"_→_"Connect Manually"_:

    <div className="flex justify-center mb-8">
        <img className="w-[40%]" src={AddAzureAISearchIndex}/>
    </div>

    - **Endpoint**: `https://srch-ai-<id>.search.windows.net`. You can get this value directly from the Azure Search resource.
    - **Auth Type**: "Azure Active Directory". This will use the managed identity attached to this resource.
    - **Connection Name**: `project-ai-<id>-conn-search`

## Create a simple agent in AI Foundry (via interface)

1. From the Foundry portal, click on the _"Build"_ tab and go to _"Agents"_ to create a new `ai-knowledge-agent` agent:

    <div className="flex justify-center mb-8">
        <img className="w-[70%]" src={CreateAgent}/>
    </div>

Use the following information:

- **Model**: Select the previously deployed `gpt-4.1 nano` model.
- **Instructions**: 

    ```plaintext
    # PURPOSE

    You are an agent specialized in AI-related questions. You help users improve their AI-related knowledge based on several specialized articles.

    # INSTRUCTIONS		
            
    - Provide detailed answers useful for practitioners. Avoid generic responses.
    - **All** inquiries seeking information require a search.
    - You **must never** rely on internal knowledge alone; searches **must** be performed every time users request information.
    - If no relevant information is found in the 'ai-knowledge-index' index, you **must explicitly state that you cannot answer the question**.
    - Your responses **must be strictly based on retrieved information**.
    - You **must not** speculate, infer, assume, or provide external knowledge beyond what is contained within 'ai-knowledge-index'.
    - **Dates, times, historical details, or contextual assumptions must never be modified**.
    - **There are no exceptions** to performing searches when the user seeks information.
    - EVERY answer must ALWAYS provide citations when using the Azure AI Search tool and render them as: "【message_idx:search_idx†source_name】" replacing values dynamically with values from context.
    - Make your answers concise.

    # WORKFLOW

    - When asked about AI topics:
        - If the question relates to AI article data, **ALWAYS** retrieve relevant data using the 'ai-knowledge-index' search index.
        - Answer the question based on retrieved data.
    ```

1. On the _"Tools"_ section, add a new tool and select _"Azure AI Search"_:

    <div className="flex justify-center mb-8">
        <img className="w-[70%]" src={AISearchToolConnection}/>
    </div>

1. Click on _"Connect to a new resource"_ and select the previously created connection to your Azure AI Search resource: 

    <div className="flex justify-center mb-10">
        <img className="w-[35%] m-5" src={CreateNewResourceAISearchTool}/>  
        <img className="w-[65%] m-5" src={SelectAISearchConnection}/>
    </div>

1. Select the `ai-knowledge-index` index in the list and click on _"Add"_:

    <div className="flex justify-center mb-8">
        <img className="w-[70%]" src={SelectAISearchIndex}/>
    </div>

1. On the tool parameters, set the query type to **Vector**:

    <div className="flex justify-center mb-8">
        <img className="w-[70%]" src={VectorSearchQueryConfig}/>
    </div>


1. On the playground chat, test your agent with the following query: 

    ```plaintext
    How to build a neural network?
    ```

    <div className="flex justify-center mb-8">
        <img className="w-[70%]" src={TestAgentFoundry}/>
    </div>

 
    If the agent answers it correctly based on the indexed data including citations, congratulations! You have successfully created a simple RAG agent using Microsoft Foundry and Azure AI Search.

- Implemented strict guidelines ensuring responses are based solely on retrieved data
- Added citation requirements to provide transparency and traceability in agent responses
- Emphasized the importance of search operations for every information request

**Azure AI Search Tool Setup**
- Integrated the Azure AI Search tool into your agent's capabilities
- Connected the tool to your AI Search resource using the previously created connection
- Selected the `ai-knowledge-index` as the knowledge source for your agent
- Configured vector search as the query type for semantic understanding and better retrieval accuracy
- Enabled the agent to perform intelligent searches across your indexed AI knowledge articles

- Tested your agent using the AI Foundry playground with sample queries
- Verified that the agent correctly retrieves information from indexed data
- Confirmed that responses include proper citations using the Azure AI Search tool
- Validated the RAG pattern: retrieving relevant information before generating responses
- Ensured the agent follows the configured instructions and workflow
